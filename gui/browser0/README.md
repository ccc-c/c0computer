# browser0 - 輕量級Markdown本地瀏覽器

## 簡介
browser0 是一款基於 C 語言（C99 標準）開發的**輕量級本地 Markdown 文件瀏覽器**，專為 macOS（ARM 架構/M1/M2/M3）設計。無需依賴網頁瀏覽器或覆雜的 Markdown 解析庫，通過 SDL2 直接渲染本地 Markdown 文件，聚焦「輕量、高效、本地運行」核心需求，適配 Retina 高分辨率屏幕，支持基礎的文本瀏覽和鏈接跳轉功能。

## 核心特性
- 🎈 **極致輕量**：純本地 C 語言開發，無網頁引擎、無多餘依賴，僅依賴 SDL2/SDL2_ttf 基礎圖形庫；
- 📝 **基礎 Markdown 支持**：渲染純文本 Markdown 內容，解析並支持 `[鏈接文本](路徑)` 格式的本地文件跳轉；
- 🖥️ **高DPI適配**：針對 macOS Retina 屏優化渲染邏輯，解決文字模糊、界面拉伸問題；
- 🖱️ **友好交互**：支持鼠標滾輪滾動瀏覽、窗口大小調整、點擊本地鏈接跳轉文件；
- 🇨🇳 **中文字體適配**：默認適配 macOS 系統中文字體（蘋方/黑體），保證中文渲染清晰；
- ⚡ **硬件加速**：基於 SDL2 硬件渲染器，文本渲染流暢無卡頓。

## 技術棧
| 技術/庫       | 作用                          |
|---------------|-------------------------------|
| C99           | 核心開發語言，保證輕量高效    |
| SDL2          | 窗口管理、圖形渲染、事件處理  |
| SDL2_ttf      | 字體加載與文本渲染            |
| Homebrew      | macOS 下依賴庫安裝與管理      |

## 快速開始
### 1. 環境準備（macOS）
確保已安裝 SDL2 和 SDL2_ttf 依賴：
```bash
brew install sdl2 sdl2_ttf
```

### 2. 編譯項目
將項目文件放在同一目錄，執行編譯腳本：
```bash
# 賦予腳本執行權限（首次運行）
chmod +x test.sh
# 編譯項目
./test.sh
```

### 3. 運行瀏覽器
```bash
# 格式：./browser0 本地Markdown文件路徑
./browser0 md/index.md
```

## 功能說明
### 基礎操作
- **窗口交互**：支持拖動窗口邊緣調整大小，窗口大小改變時文本自動重新排版；
- **文本瀏覽**：鼠標滾輪上下滾動，瀏覽超出窗口高度的長文本；
- **鏈接跳轉**：點擊 Markdown 中的本地鏈接（相對/絕對路徑），自動加載並渲染目標文件；
- **退出程序**：點擊窗口關閉按鈕（×）即可退出，自動釋放資源。

### 渲染範圍
當前版本聚焦「純文本 Markdown」核心需求，暫不支持以下格式（後續可擴展）：
- 標題層級、加粗/斜體等富文本樣式；
- 圖片、表格、代碼塊等覆雜元素；
- 遠程網絡鏈接（僅支持本地文件路徑）。

## 注意事項
1. 僅適配 macOS ARM 架構（M1/M2/M3），Intel 架構需微調編譯參數；
2. 依賴 macOS 系統默認中文字體，若系統字體缺失需手動調整代碼中的字體路徑；
3. 僅支持本地 Markdown 文件，暫不支持遠程文件或網絡鏈接；
4. 窗口默認尺寸為 600×450（邏輯像素），Retina 屏自動渲染為 1200×900 物理像素，可在 `md0render.h` 中修改 `WINDOW_WIDTH`/`WINDOW_HEIGHT` 調整。

## 適配與優化
- 若文字尺寸不合適：修改 `md0render.h` 中的 `FONT_SIZE` 常量（默認 24，可調整為 20/28）；
- 若邊距過大/過小：調整 `md0render.c` 中 `get_margin_px` 函數的邊距比例（默認 3%）；
- 若高DPI渲染異常：修改 `get_window_scale` 函數的返回值（Retina 屏設為 2.0，普通屏設為 1.0）。
